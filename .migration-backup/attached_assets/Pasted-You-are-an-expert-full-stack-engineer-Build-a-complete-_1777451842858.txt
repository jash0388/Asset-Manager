You are an expert full-stack engineer. Build a complete production-ready mobile-first web + Android-compatible application called **"Smart QR Attendance, Access Control & Student Tracking System"**.

This system is used in a college where a security guard scans ID card QR codes to automatically mark attendance (entry/exit), while admins can search any student and view their complete past attendance history.

---

## 🎯 CORE SYSTEM MODULES

Build a complete system with:

1. Mobile QR Scanner App (for security guards)
2. Backend API server
3. Admin Dashboard (Web)
4. Search & History Tracking System (IMPORTANT)
5. Authentication & Security
6. Database (persistent storage)

---

## 🧱 TECH STACK (STRICT)

* Frontend (Dashboard + Scanner UI): React + Tailwind CSS
* Mobile-friendly Scanner: React PWA (camera access)
* Backend: Node.js + Express
* Database: MongoDB with Mongoose
* QR Scanner: html5-qrcode (or similar)
* Authentication: JWT-based login

### 📷 4. SCANNER SYSTEM (MAIN FEATURE)

* Mobile responsive full-screen scanner

* Camera-based QR scanning

* On scan:

  * Send QR ID to backend
  * Validate user
  * Mark attendance

* Show instant UI feedback:

  * ✅ Access Granted (with name)

### ⏱️ 5. ATTENDANCE LOGIC (STRICT)

* If first scan of the day → mark ENTRY
* If second scan → mark EXIT
* If more scans → ignore or log safely

Store:

* userId
* date (YYYY-MM-DD)
* entryTime
* exitTime
* scanCount

Prevent misuse:

* Ignore repeated scans within 30 seconds

### 🌐 8. API ENDPOINTS

Auth:

* POST /api/auth/login

Users:

* POST /api/users
* GET /api/users
* GET /api/users/:id
* DELETE /api/users/:id

QR:

* GET /api/qrcode/:userId

Attendance:

* POST /api/scan
* GET /api/attendance
* GET /api/attendance/today
* GET /api/attendance/user/:userId  ← IMPORTANT (history endpoint)

Search:

* GET /api/search?query=NAME_OR_ID
